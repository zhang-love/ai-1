# 客服培训机器人 - 技术流程说明

## 📋 整体架构

```
用户 ↔ 前端界面 ↔ FastAPI后端 ↔ Claude API (火山引擎)
                    ↓
                  知识库 (Markdown)
```

---

## 🎯 完整流程（优化版）

### 📨 步骤一：用户输入 → 后端接收

#### 前端动作
1. 用户在聊天框输入消息
2. 按回车或点击"发送"
3. 前端通过 **POST 请求** 发送到 `/api/chat`

#### 请求数据格式
```json
{
  "session_id": "当前会话ID",
  "message": "用户输入的内容",
  "mode": "free"  // free:自由问答, scenario:场景模拟
}
```

#### 后端接收
FastAPI 接收请求，解析 JSON 数据

---

### 🛡️ 步骤二：后端预处理

#### 2.1 会话管理
- 查找或创建用户会话
- 将用户消息存入会话历史
- 历史记录限制：最多保留 20 条（避免 context 过长）

#### 2.2 加载知识库
- 读取 `knowledge/` 目录下的 Markdown 文件
- 支持多文件合并
- 格式：产品知识 + 常见问题

#### 2.3 组装提示词 (Prompt Engineering)

##### 系统提示词模板
```
你是一个专业的客服培训助手。你的任务是帮助新客服快速掌握产品知识和服务技巧。

以下是产品知识库，请基于这些知识回答问题：

{知识库内容}

## 你的角色设定
- 友好、专业、耐心
- 回答要基于知识库内容，不要编造信息
- 如果知识库中没有答案，诚实告知
- 可以参考知识库中的客服话术规范
```

##### 场景模拟模式（AI 扮演客户）
```
你现在需要扮演一个客户，与客服进行对话练习。

## 当前场景
{场景描述：如产品连接问题}

## 客户特征
{如：新手用户，不太懂技术，比较着急}

## 产品知识库（供你参考）
{知识库内容}

## 扮演要求
- 用客户的语气说话
- 不要暴露你是 AI 的身份
- 表现得像一个真实的客户
```

---

### 🌐 步骤三：调用 Claude API

#### 3.1 构造 API 请求

**请求 URL**
```
https://ark.cn-beijing.volces.com/api/coding/v1/messages
```

**请求头**
```http
Content-Type: application/json
Authorization: Bearer {API_KEY}
anthropic-version: 2023-06-01
```

**请求体**
```json
{
  "model": "ark-code-latest",
  "max_tokens": 4096,
  "system": "系统提示词内容",
  "messages": [
    {"role": "user", "content": "用户问题1"},
    {"role": "assistant", "content": "AI回答1"},
    {"role": "user", "content": "用户问题2"}
  ]
}
```

#### 3.2 错误处理

| 错误类型 | 处理方式 |
|---------|---------|
| 超时 (Timeout) | 显示"请稍候"，最多重试 2 次 |
| 认证失败 (401) | 检查 API Key 配置 |
| 路径错误 (404) | 使用演示模式回退 |
| 其他错误 | 记录日志，提示用户稍后再试 |

---

### 💬 步骤四：处理并返回响应

#### 4.1 解析 API 响应

**成功响应示例**
```json
{
  "id": "msg_xxx",
  "content": [
    {
      "type": "text",
      "text": "AI生成的回复内容"
    }
  ],
  "usage": {
    "input_tokens": 2048,
    "output_tokens": 512
  }
}
```

#### 4.2 提取回复
从响应中提取 `content[0].text` 作为最终回复

#### 4.3 存入会话历史
将 AI 回复添加到会话消息列表

#### 4.4 返回前端

**响应格式**
```json
{
  "response": "AI的回复内容",
  "status": "ok",
  "demo": false  // 是否为演示模式
}
```

## 🔄 演示模式回退（重要）

当 API 不可用时，自动回退到演示模式：

```python
if api_available:
    try:
        # 尝试调用 API
        reply = api_client.chat(messages, system_prompt)
    except Exception:
        # 失败，使用演示回复
        reply = get_demo_response(user_msg)
else:
    # API 不可用，直接用演示回复
    reply = get_demo_response(user_msg)
```

**演示回复规则**
- 关键词匹配（如"蓝牙"、"连接"）
- 预设场景回复
- 默认通用回复

## 📁 项目模块分工

| 模块 | 文件 | 职责 |
|------|------|------|
| API 客户端 | `api_client.py` | 封装 Claude API 调用，处理认证和重试 |
| 知识库 | `knowledge.py` | 加载、管理 Markdown 知识库 |
| 会话管理 | `chat_manager.py` | 管理用户会话和历史消息 |
| 提示词 | `prompts.py` | 生成系统提示词模板 |
| 主服务 | `main.py` | FastAPI 路由、请求处理 |

---

## 🔐 安全最佳实践

1. **API Key 保护**
   - 不提交到 Git
   - 环境变量加载
   - .env 文件只在本地

2. **内容安全**
   - 敏感词过滤（可选）
   - 用户消息长度限制

3. **会话隔离**
   - 每个用户独立 session
   - 会话 ID 用 UUID 生成
