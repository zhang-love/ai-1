# 产品客服培训机器人 - 项目文档

## 项目背景

参考飞智科技的 Agent 开发思路，打造一个基于 Claude 的产品客服培训系统。

### 目标产品领域
- 产品：游戏手柄、消费电子（参考飞智科技）
- 场景：新客服入职培训、产品知识考核、服务话术演练

---

## 版本规划

### 版本一：基础原型（MVP）
**目标：** 快速验证核心功能，可在 1-2 小时内完成

#### 功能清单

| 模块 | 功能 | 优先级 |
|------|------|--------|
| 📚 知识库 | 产品知识 Markdown 导入 | P0 |
| 💬 对话训练 | 单轮问答、基于产品知识回答 | P0 |
| 🎯 场景模拟 | 常见客服场景演练 | P1 |
| 📝 简单测评 | 基础答题功能 | P1 |

#### 技术选型

```
前端：HTML + CSS + JavaScript（单文件应用）
后端：Python + FastAPI
AI：Claude API（通过火山引擎）
存储：本地 JSON 文件
```

#### 项目结构

```
ai-1/
├── README.md              # 项目说明
├── 项目规划.md            # 本文档
├── 需求文档.md            # 详细需求（含代码实现要点）
├── CLAUDE.md              # Claude Code 指导
├── knowledge/             # 知识库目录
│   └── 产品知识库.md      # 产品知识
├── src/
│   ├── main.py            # 后端主文件
│   ├── api_client.py      # API 封装
│   ├── chat_manager.py    # 对话管理
│   ├── knowledge.py       # 知识库加载
│   ├── prompts.py         # Prompt 模板
│   └── templates/
│       └── index.html     # 前端界面
├── requirements.txt       # Python 依赖
└── .env.example           # 配置示例
```

#### 代码关键点

**1. API 调用封装**
- 从环境变量读取 API key 和 base URL（Claude Code 会自动注入 settings.json 中的 env）
- 处理请求超时（已配置 3000000ms = 50分钟）
- 使用官方 anthropic Python SDK

**2. 知识库加载**
- Markdown 文件解析
- 支持多文件加载
- 简单的文本分块策略

**3. Prompt 设计**
- 系统提示词模板
- 知识库内容注入
- 角色设定（客服培训助手）

**4. 对话管理**
- 会话历史管理
- 上下文窗口控制
- 消息格式化

**5. Web 服务**
- FastAPI RESTful API 设计
- 静态文件服务
- Jinja2 模板渲染

#### 核心实现思路

1. **知识库加载**：读取 Markdown 文件，作为上下文
2. **Prompt 设计**：
   ```
   你是一个专业的客服培训助手。基于以下产品知识回答用户问题：

   [产品知识]

   请用专业、友好的客服语气回答。
   ```
3. **对话界面**：简单的聊天 UI
4. **场景模式**：预设几种常见客服场景
