# 📚 产品知识库使用说明

## 🎯 一句话总结

知识库通过**"加载文件 → 注入提示词 → 传给 Claude API"**的流程，让 AI 基于你的产品知识回答问题！

---

## 📖 详细流程

### 1️⃣ 第一步：加载知识库

**文件**: `src/knowledge.py`

```python
# 项目启动时执行
knowledge_manager = KnowledgeManager()

# 做了这些事情：
# 1. 扫描 knowledge/ 目录
# 2. 读取所有 .md 文件
# 3. 把内容存入 self.knowledge_cache

# knowledge_cache 的结构：
{
    "产品知识库.md": "完整的 Markdown 内容..."
}
```

---

### 2️⃣ 第二步：组装提示词

**文件**: `src/prompts.py`

```python
# 当用户问 "我的手柄连不上蓝牙怎么办？" 时：

system_prompt = f"""你是一个专业的客服培训助手。

以下是产品知识库，请基于这些知识回答问题：

# 飞智游戏手柄 - 产品知识库

## 常见问题

### Q: 手柄如何连接蓝牙？
A: 您好！蓝牙连接步骤如下...

### Q: 手柄连不上蓝牙怎么办？
A: 您好！请先检查以下几点...

## 你的角色设定
- 友好、专业、耐心
- 回答要基于知识库内容，不要编造信息
- ...
"""
```

注意：整个 Markdown 文件会**直接插入到提示词中**！

---

### 3️⃣ 第三步：调用 Claude API

**文件**: `src/api_client.py` 或 `api_client_http.py`

```python
# 发送给 Claude 的请求：
{
    "model": "claude-opus-4-8-1m",
    "system": "上面的那个长提示词（包含知识库）",
    "messages": [
        {"role": "user", "content": "我的手柄连不上蓝牙怎么办？"}
    ],
    "max_tokens": 4096
}
```

---

### 4️⃣ 第四步：返回结果

Claude 基于知识库内容回答，返回给用户！

---

## 💡 知识库文件格式建议

### ✅ 当前格式（很好）

```markdown
# 飞智游戏手柄 - 产品知识库

## 产品系列
- 八爪鱼系列
- 黑武士系列

## 常见问题

### Q: 手柄如何连接蓝牙？
A: 您好！蓝牙连接步骤如下...
```

### 📝 格式优势

1. **Markdown 易读易写**
2. **Q&A 结构清晰**
3. **可以直接插入提示词**

## 🔧 如何添加新知识

### 方法一：编辑现有文件
直接修改 `knowledge/产品知识库.md`，重启服务即可生效！

### 方法二：添加新文件
创建 `knowledge/新产品知识.md`，系统会自动加载所有 `.md` 文件！

```python
# knowledge.py 中的这段代码会合并所有文件：
def get_knowledge_content(self, filename=None):
    if filename and filename in self.knowledge_cache:
        return self.knowledge_cache[filename]
    return "\n\n".join(self.knowledge_cache.values())
```

## 🤔 常见问题

### Q: 知识库内容太长，会不会超 token 限制？
**A**: 当前是一次性全部注入，可以优化为：
- 按产品分文件
- 使用 RAG 检索相关内容

### Q: 可以实时更新知识库吗？
**A**: 目前需要重启服务。后续可以添加 `reload` 接口！

### Q: 知识库中的格式重要吗？
**A**: 只要是可读文本就行！Markdown、纯文本都可以。

## 📊 数据流向图

```
产品知识库.md
    ↓ (加载)
knowledge_cache['产品知识库.md']
    ↓ (获取)
knowledge_content = "完整的 Markdown..."
    ↓ (注入)
system_prompt = "你是客服助手...\n\n" + knowledge_content
    ↓ (发送)
Claude API
    ↓ (返回)
AI 回答
    ↓ (展示)
用户界面
```
